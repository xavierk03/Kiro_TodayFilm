## 1. 제품 개요 & 비전

### 1.1 한 줄 소개
> **사진과 AI 대화를 통해 하루를 돌아보고, 시간이 지나면 과거와 현재의 기억을 연결해 주는 AI 회고 서비스**

### 1.2 서비스 개요
오늘의 필름은 하루를 마무리하며 오늘의 사진을 선택하고, AI와 짧은 대화를 나누며 하루를 돌아보는 서비스입니다. AI는 사용자의 감정을 대신 판단하거나 분석하지 않고, 사진과 사용자의 답변을 바탕으로 질문을 이어가며 사용자가 스스로 하루를 돌아보도록 돕습니다. 회고가 끝나면 사진과 대화를 바탕으로 하루의 기록이 하나의 **필름**으로 완성됩니다. 기록이 쌓일수록 AI는 과거의 고민·목표·관계와 현재의 순간을 연결하여, 사용자가 시간에 따른 자신의 변화를 발견하도록 합니다.

### 1.3 기획 배경 (Background)
- 우리는 매일 사진을 찍지만, 사진은 **장면만 남길 뿐** 왜 찍었는지·무슨 생각을 했는지는 남기지 못한다. 시간이 지나면 그 안의 이야기는 흐려진다.
- 일기는 생각까지 남기지만 **긴 글을 써야 하는 부담**이 있고, 평범한 하루엔 무엇을 쓸지 막막해 지속이 어렵다.
- 최근 AI 기록 서비스는 부담을 줄여주지만, 대부분 사용자의 글·사진을 **분석해 감정을 분류·요약**하는 방식에 집중한다.
- 오늘의 필름의 출발 질문:
  > **"AI가 나의 하루를 대신 분석하는 것이 아니라, 내가 스스로 하루를 돌아볼 수 있도록 도와줄 수는 없을까?"**

### 1.4 문제 정의 (Problem Definition)
| # | 문제 | 설명 |
|---|------|------|
| P1 | **사진은 남지만, 그날의 이야기는 사라진다** | 사진은 장면만 보여줄 뿐 왜 찍었는지·당시 생각은 기록되지 않아, 시간이 지나면 개인적 기억이 흐려진다. |
| P2 | **기록을 꾸준히 이어가기 어렵다** | 일기는 직접 정리해 글로 써야 하는 부담이 크고, 평범한 하루엔 무엇을 쓸지 막막해 중단하게 된다. |
| P3 | **기록은 쌓이지만 서로 연결되지 않는다** | 사진·일기는 날짜별로 저장될 뿐, 과거의 고민이 지금 어떻게 달라졌는지 같은 시간에 따른 변화는 직접 찾지 않으면 발견하기 어렵다. |

### 1.5 프로젝트 목표 (Project Goal)
> **기록을 통해 자신의 시간과 변화를 발견하는 경험**을 만든다.

| # | 목표 | 대응 문제 |
|---|------|-----------|
| G1 | **기록의 부담을 낮춘다** — 긴 글 없이 사진 + 짧은 AI 대화만으로 회고 | P2 |
| G2 | **스스로 하루를 돌아보게 한다** — 감정 판단 대신 질문으로 유도 | P1 |
| G3 | **흩어진 기억을 연결한다** — 과거와 현재의 연결점으로 변화 발견 | P3 |
| G4 | **지속하고 싶은 경험으로 만든다** — 회고를 '필름'으로 완성·아카이빙해 시각적으로 쌓이게 | P2 |

### 1.6 차별점
- **판단하지 않는 AI**: 감정 단정·사실 추측 금지, 질문으로만 회고를 유도.
- **사진 기반 시작점**: 빈 화면 대신 오늘 찍은 사진에서 대화 시작.
- **시간 축 연결(구조적 차별점)**: 과거 "기억 포인트"와 의미 유사도로 연결해 변화를 발견.

## 3. 범위 (Scope)

### 3.1 In Scope `[MVP]`
- 오늘 찍은 사진 1~3장 업로드 + 대표 사진 지정
- AI 회고 대화: 시작 → 확장 → 마무리의 **3문 고정 구조**
- AI 결과물: 오늘의 제목 / 한 줄 회고 / 대화 전체 / 필름 카드 1종
- 자동 저장 및 사용자별 격리 보관
- 월간 캘린더 아카이브 (날짜별 기록 확인 + 상세)

### 3.2 In Scope `[확장]`
- 사진 4~5장 업로드
- 기록 패턴 기반 개인화 질문
- 과거 기록 연결 (기억 포인트 + pgvector 유사도)
- 시각 결과물 다양화(포스터 등), 월간·연간 아카이브, 키워드별 모아보기

### 3.3 Out of Scope (이번 프로젝트에서 하지 않음)
- 감정 점수화 / 감정 분석 대시보드 (제품 철학과 배치)
- 소셜 피드 / 타 사용자와의 공개 공유 (기본 비공개, 대화 내역은 공유 제외)
- 실시간 협업, 멀티 유저 편집
- 웹 클라이언트 `[가정]` (Flutter 모바일 우선)

### 3.4 데모/발표 고려사항
- 과거 기록 연결은 **기록 누적이 전제**이므로, 발표용으로 **사전 준비된 과거 기록 1건**으로 연결 시나리오를 안정적으로 시연할 최소 구현 권장.

## 4. 기능 요구사항 (Functional Requirements)

주신 6개 핵심 기능을 요구사항으로 재정리했습니다.

### FR-1. 사진 업로드 및 선택
- FR-1.1 오늘 촬영한 사진을 EXIF 촬영 날짜 기준으로 자동 추천한다.
- FR-1.2 갤러리에서 직접 선택할 수 있다.
- FR-1.3 1~5장 선택 가능하며, 2장 이상일 때 대표 사진 1장을 지정한다. `[MVP: 1~3장]`
- FR-1.4 첫 질문은 대표 사진 기준으로 생성한다.
- FR-1.5 업로드 시 이미지를 리사이즈·압축한다.
- FR-1.6 서버 저장 전에 EXIF 위치 정보를 제거한다.

### FR-2. AI 회고 대화
- FR-2.1 Vision 모델이 대표 사진의 맥락(장면/사물/분위기)을 파악해 첫 질문을 생성한다.
- FR-2.2 사진 맥락은 사용자에게 노출하지 않고 질문에만 녹인다.
- FR-2.3 사용자 답변의 핵심 표현을 이어받아 후속 질문을 생성한다.
- FR-2.4 질문은 최대 3개, 전체 대화는 2~3분 내로 설계한다.
- FR-2.5 모든 질문은 건너뛸 수 있다.
- FR-2.6 언제든 종료 가능하며, 그때까지의 답변으로 기록을 생성한다.
- FR-2.7 AI 대화 원칙(감정 단정 금지, 사실 추측 금지)을 시스템 프롬프트로 고정한다.

### FR-3. AI 결과물 생성
- FR-3.1 입력은 선택 사진 + 사진 맥락 + 전체 대화이다.
- FR-3.2 출력은 오늘의 제목 / 한 줄 회고 / 대화 전체 / 필름 카드이다.
- FR-3.3 LLM 구조화 출력(JSON: title / one_line / quote)을 사용한다.
- FR-3.4 사용자의 표현을 우선 재료로 쓰고, 대화에 없던 감정 해석을 추가하지 않는다.
- FR-3.5 사용자는 제목과 한 줄 회고를 수정할 수 있다.
- FR-3.6 필름 카드를 렌더링한다(Flutter RepaintBoundary 또는 서버 Pillow 템플릿). `[MVP: 템플릿 1종]`

### FR-4. 기록 저장 및 보관
- FR-4.1 회고 종료 시 별도 버튼 없이 자동 저장한다(유실 방지).
- FR-4.2 저장 단위는 날짜 + 사진 + 대화 전체 + 제목/한 줄 회고 + 필름 카드이다.
- FR-4.3 기본 비공개로 저장하고, 대화 내역은 공유 대상에서 제외한다.
- FR-4.4 사용자별로 데이터를 격리한다.

### FR-5. 기록 아카이브
- FR-5.1 월간 캘린더 뷰에서 기록이 있는 날짜에 대표 사진 썸네일을 표시한다.
- FR-5.2 날짜 선택 시 상세 화면에서 원본 사진 / 그날의 대화 / AI 결과물을 함께 보여준다.

### FR-6. 과거 기록 연결 `[확장]`
- FR-6.1 회고 종료 시 대화에서 고민/목표/관계/기대에 해당하는 표현을 "기억 포인트"로 구조화해 저장한다.
- FR-6.2 새 기록 생성 시 사진 맥락·답변을 기존 기억 포인트와 임베딩 유사도로 비교한다.
- FR-6.3 임계값 이상일 때만 대화 중 1회(시간 연결형 질문) 또는 결과 화면의 '그때의 기록' 카드로 제시한다.
- FR-6.4 회고당 최대 1회만 연결을 제시한다(과도한 회상 방지).
- FR-6.5 사용자는 특정 기록의 회상을 차단할 수 있다.
- FR-6.6 애매한 연결은 제시하지 않는다(정밀도 우선).

## 6. 유저 스토리 & 수락 기준

User Flow(7장)를 스토리로 옮기고, EARS 스타일 수락 기준을 붙였습니다.

### US-1. 회고 시작
> 사용자로서, 홈에서 한 번의 탭으로 오늘의 회고를 시작하고 싶다. 기록을 시작하는 마찰을 줄이기 위해서.

- WHEN 사용자가 홈에서 "오늘 하루, 돌아볼까요?"를 탭하면 THEN 시스템은 사진 선택 단계로 진입한다.
- IF 오늘 이미 기록이 있으면 THEN 시스템은 새 회고 대신 해당 기록을 보여준다.

### US-2. 사진 선택
> 사용자로서, 오늘 찍은 사진 중에서 골라 회고의 단서로 삼고 싶다.

- WHEN 사진 선택 화면에 진입하면 THEN 시스템은 오늘 촬영 사진(EXIF 기준)을 자동 추천한다.
- 사용자는 갤러리에서 직접 선택할 수 있다(SHALL).
- WHEN 사용자가 2장 이상 선택하면 THEN 시스템은 대표 사진 1장 지정을 요구한다.
- WHEN 사진이 업로드되면 THEN 시스템은 리사이즈·압축하고 EXIF 위치정보를 제거한 뒤 저장한다.

### US-3. AI 첫 질문
> 사용자로서, 사진을 고른 이유를 스스로 떠올리게 하는 열린 질문을 받고 싶다.

- WHEN 대표 사진이 정해지면 THEN 시스템은 사진 맥락 기반 개방형 질문을 1개 생성한다.
- 시스템은 사진 속 상황을 단정하는 표현을 사용하지 않아야 한다(SHALL NOT).
- 시스템은 사진 맥락 정보를 사용자에게 직접 노출하지 않아야 한다(SHALL NOT).

### US-4. 대화 진행
> 사용자로서, 내 답변을 이어받는 후속 질문을 받고, 얼마나 남았는지 알고 싶다.

- WHEN 사용자가 답변하면 THEN 시스템은 답변의 핵심 표현을 이어받아 후속 질문을 생성한다.
- 시스템은 남은 질문 수를 표시해야 한다(SHALL).
- IF 답변이 짧으면 THEN 더 가벼운 질문을, 길고 구체적이면 한 단계 깊은 질문을 생성한다.
- 사용자는 어떤 질문이든 건너뛸 수 있다(SHALL).

### US-5. 회고 종료
> 사용자로서, 원할 때 언제든 회고를 끝낼 수 있어야 한다.

- WHEN 질문 3개가 소진되면 THEN 시스템은 결과물 생성으로 넘어간다.
- WHEN 사용자가 "여기까지 할게요"를 누르면 THEN 시스템은 즉시 종료하고 결과물 생성으로 넘어간다.
- WHEN 사용자가 2회 연속 건너뛰면 THEN 시스템은 "오늘은 여기까지 할까요?"를 제안한다.

### US-6. 결과물 확인 및 수정
> 사용자로서, 하루가 정리된 결과물을 받고 필요하면 다듬고 싶다.

- WHEN 회고가 종료되면 THEN 시스템은 제목 / 한 줄 회고 / 필름 카드를 생성한다.
- 시스템은 대화에 없던 감정·해석을 추가하지 않아야 한다(SHALL NOT).
- 사용자는 제목과 한 줄 회고를 수정할 수 있다(SHALL).

### US-7. 자동 저장 & 아카이브
> 사용자로서, 내가 만든 기록이 유실 없이 쌓이는 걸 보고 싶다.

- WHEN 사용자가 결과물을 확인하면 THEN 시스템은 즉시 자동 저장한다.
- WHEN 사용자가 캘린더의 날짜를 선택하면 THEN 시스템은 그날의 사진·대화·결과물을 함께 보여준다.

### US-8. 과거 기록 연결 `[확장]`
> 사용자로서, 지금의 나와 과거의 나를 연결해 변화를 발견하고 싶다.

- WHEN 새 기록이 생성되면 THEN 시스템은 기억 포인트와의 유사도를 계산한다.
- IF 유사도가 임계값 이상이면 THEN 시스템은 회고당 최대 1회 연결을 제시한다.
- IF 연결할 과거 기록이 없으면 THEN 시스템은 연결 기능을 노출하지 않는다.
- 사용자는 특정 기록을 "다시 꺼내지 않기"로 설정할 수 있다(SHALL).

### 예외 흐름 요약 (7장 기반)
| 상황 | 처리 |
|------|------|
| 여러 장 업로드 | 대표 사진으로 첫 질문, 나머지는 필름 카드에 함께 배치. 후속 질문에서 다른 사진 언급 가능 |
| 답변 건너뛰기 | 다음 질문으로. 2회 연속이면 조기 종료 제안 |
| 중간 종료 | 그때까지 답변으로 결과물 생성 |
| 답변 0개 | 사진+날짜만으로 최소 기록(제목 후보 포함) 저장, 나중에 이어서 회고 |
| 연결 대상 없음 | 연결 기능 미노출. 신규 사용자 경험은 연결 없이도 완결 |
| 사용자 무반응 | 제시한 연결을 결과물에 포함하지 않음 |

## 7. 도메인 모델

### 7.1 핵심 엔티티
| 엔티티 | 설명 | 주요 속성 |
|--------|------|-----------|
| **User** | 서비스 사용자 | id, auth_id, created_at |
| **Film (기록)** | 하루치 기록 단위. 아카이브의 최소 단위 | id, user_id, date, title, one_line, visibility(기본 private), created_at |
| **Photo** | 필름에 속한 사진 | id, film_id, storage_path, is_representative, order, exif_removed |
| **Conversation** | 한 회고의 대화 세션 | id, film_id, status(진행/종료), ended_reason, question_count |
| **Message** | 대화의 개별 질문/답변 | id, conversation_id, role(ai/user), content, is_skipped, seq |
| **PhotoContext** | Vision 분석 결과(비노출) | id, photo_id, scene, objects, mood (사용자 비노출) |
| **FilmCard** | 렌더된 결과물 카드 | id, film_id, template_id, image_path |
| **MemoryPoint** `[확장]` | 회고에서 추출한 구조화 표현 | id, user_id, film_id, type(고민/목표/관계/기대), text, embedding(vector), topic_tag, is_blocked |
| **MemoryLink** `[확장]` | 새 기록과 과거 기억 포인트의 연결 | id, film_id, memory_point_id, similarity, presented_as(질문/카드), user_reacted |

### 7.2 관계 (개요)
```
User (1) ──< (N) Film
Film (1) ──< (N) Photo            (Photo.is_representative = 대표 1장)
Film (1) ─── (1) Conversation
Conversation (1) ──< (N) Message
Photo (1) ─── (1) PhotoContext    (비노출, 질문 생성용)
Film (1) ─── (1) FilmCard
User (1) ──< (N) MemoryPoint       [확장]
Film (1) ──< (N) MemoryLink >── (1) MemoryPoint   [확장]
```

### 7.3 핵심 불변식 (Invariants)
- 한 사용자는 하루(date)에 하나의 Film을 가진다. **(MVP 확정: 하루 1기록, `(user_id, date)` 유니크)**
- 사진이 2장 이상이면 정확히 1장이 대표 사진이다.
- 답변이 0개여도 Film은 저장된다(최소 기록).
- PhotoContext는 질문 생성에만 쓰이고 사용자에게 직접 노출되지 않는다.
- MemoryPoint가 차단(is_blocked)이면 연결 후보에서 제외된다.
